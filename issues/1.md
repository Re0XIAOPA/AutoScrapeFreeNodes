### 🐞 Bug 描述  
#### 控制台倒计时没有实时效果  
在控制台中，倒计时显示为“24小时倒计时”，该功能无法实时刷新，每次操作后仍然保持初始状态，没有进行预期倒计时更新。  
#### API 网站检测多次触发  
通过 API 检测网站在线状态时，首次打开时检测到的状态是正常的。但是，在未刷新页面的情况下，如果直接关闭 API 界面后再重新打开第二次检测，状态会重新触发，而且检测的记录会出现双倍累加的问题。如果反复进行相同的操作，检测结果也会以此类推呈现指数累加。  
---  
### 🚶‍♂️ 复现步骤  
该问题可通过以下步骤复现：  
1. 打开项目，并进入控制台观察倒计时显示。  
2. 注意倒计时的刷新状态，无法实时更新。  
3. 在 API 页面侧，请尝试首次访问并触发网站在线状态检测，观察结果。  
4. 关闭页面，再次打开 API 页面进行在线状态检测，观察检测状态。  
5. 观察检测结果显示的数量及状态。  
---  
### 🛠 期望行为  
- 倒计时功能能够每秒实时刷新，提供准确的倒计时状态。  
- API 检测程序应避免重复触发操作，保证结果展示仅与实际检测对应，不应出现累加状态。
